A) Folder structure - top

The main plugin folder contains three subfolders:
  1. Demos: very useful for getting started
  2. Documentation
  3. Socialmix: this folder contains all plugin files
    • assets: external library
    • css: compressed CSS
    • js: minified javascript
    • html: templates
    • src: Socialmix and assets source code (uncompressed JS, CSS, LESS)
    • php: back-end

A) Installation - top

The installation process is composed of two main steps: front-end installation and back-end installation.

Front-end installation

1) Load scripts and stylesheets
Make sure the socialmix.min.js is included after jQuery. 
<!--  Socialmix css  -->
<link rel="stylesheet"  type="text/css" href="/STATIC-FOLDER/socialmix/css/socialmix.min.css" />
<!--  Socialmix skin css -->
<link rel="stylesheet" type="text/css" href="/STATIC-FOLDER/socialmix/css/skins/skin_1.min.css" />

<!--  jQuery 1.7+  -->
<script type="text/javascript" src="./assets/jquery.min.js" /<script>
<!-- Include socialmix.min.js  -->
<script src="/STATIC-FOLDER/socialmix/js/socialmix.min.js"> </script>
2) Set up the html
All you need is put a socialmix container inside your html document. 
<body >
    ...
        <div class="my-timeline-container" >
        </div>
    ...
</body >
3) Call the plugin
Call the socialmix initializer function.
Please note:
- "basePath" option is required. Socialmix uses this path to load asynchronously all assets.
- "proxy" option is required only if you are using Socialmix back-end. 
<script type="text/javascript">
$(document).ready(function(){
    var options = {
        basePath : "/STATIC-FOLDER/socialmix/",
        proxy : "./../socialmix/php/get_data.php",//optional:path to socialmix Back-end
        feeds:[
            .....
        ]
     
    };
    $(".my-timeline-container").socialmix(options);
});
</script>

See option section for all available options and feeds section to configure your feeds

Back-end installation (php)

The back-end is required only if you want to use social media feeds. If you work whit custom data you don't need this installation step and you can jump to the next section.

The back-end is a php script that requires a php 5.3 version. It has only one dependency: phpfastcache that is included in "socialmix/php/vendor" folder.

To run the back-end script, follow the instructions:
1) Move contents of socialmix/php/ in your php server directory.
2) Put your API KEYS in settings.php. 

#facebook
$facebook_app_id      = "YOUR-FACEBOOK-APP-ID  ";
$facebook_app_secret  = "YOUR-FACEBOOK-APP-SECRET";

#google plus
$gplus_api_key   = "YOUR-GPLUS-API-KEY";

#youtube plus
$youtube_api_key   = "YOUR-YOUTUBE-API-KEY";

#twitter
$twitter_consumer_key = "YOUR-TWITTER-CONSUMER-KEY";
$twitter_secret_key   = "YOUR-TWITTER-SECRET-KEY";

$viemo_access_token = "YOUR-VIMEO-ACCESS-TOKEN";
3) Configure phpfastcache settings in settings.php. If you want to use filesystem cache (default option), pleas make sure the "path" option is a writable folder. For all settings see official phpfastcache documentation . 
    
#cache
$use_cache = true;
$cache_time_seconds = 60 * 60 * 3;

$config_cache = array(
    "storage"   => "files",
    "path"      =>  "",  // path to cache folder, leave it blank for auto detect
    "htaccess"      => true, // create .htaccess to protect cache folder
);     
?>
4) Now your back-end is available with a url similar to "http://your-domain/your-backend-folder/get_data.php". Use this url as value of javascript "proxy" option.

See php option section for all available options

Feeds - top

Socialmix works with different feed types. In a single timeline you can mix different types of feeds.
The feed system is extremly flexible and allow you to load specific resources and use different "id" types. In the following you can find all allowed feed types. 

<script type="text/javascript">
$(document).ready(function(){
    var options = {
        basePath : "/STATIC-FOLDER/socialmix/",
        proxy : "./../socialmix/php/get_data.php",
        feeds:[
            //get Facebook page posts
            {type : "facebook" , id : "Page Name or Page ID"},
            
            //get Facebook page events
            {type : "facebook" , id : "Page Name or Page ID" , resources: "events"}, 
            
            //get Facebook page posts + events
            {type : "facebook" , id : "Page Name or Page ID" , resources: "all"}, 
            
            //get Twitter timeline
            {type : "twitter" , id : "twitter-username" },
            
            //get Twitter searh result
            {type : "twitter" , id : "Search keyword, you can use #,@", target:"search" },
            
            //Get youtube uploaded video by channelID
            {type : "youtube" , id : "channel-id"},
            
            //Get youtube uploaded video by username
            {type : "youtube" , id : "username", target:"username"},

            //Get vimeo uploaded video by username
            {type : "vimeo" , id : "username"}

            //Get Google Plus page posts
            {type : "gplus" , id : "+pagename" },

        ]
     
    };
    $(".my-timeline-container").socialmix(options);
});
</script>

In the following table, we explain in details all allowed feed options

type id target resources
Facebook page posts and events "facebook" Page id/name - "posts", "events", "all". Default: "posts"
Twitter User Timeline "twitter" Username "username" (Default) "all" (Default)
Twitter Search "twitter" Search keyword "search" "all" (Default)
Uploaded video by channel ID "youtube" Channel id "channelid" (Default) "all" (Default)
Uploaded video by Username "youtube" Username "username" "all" (Default)
Google plus page posts "gplus" Page id/name - "all" (Default)
Vimeo Uploaded video "vimeo" Vimeo Username - "all" (Default)
JSON custom data "custom" JSON Url - "all" (Default)
JS custom data "custom_object" Variable Name - "all" (Default)

  • * type: "facebook" | "twitter" | "gplus" | "custom"

F) Php options - top

Php options are located in settings.php file.

Name Type Description
$facebook_app_id String Facebook App ID
$facebook_app_secret String Facebook App Secret
$gplus_api_key String Google Plus Api Key
$twitter_consumer_key String Twitter Consumer Key
$twitter_secret_key String Twitter Secret Key
$csrf_protection Boolean Use csrf protecion

$config_cache array Cache parameters configuration. Default configurations use file system cache and autodetect cache folder. To change default cache settings see official phpfastcache documentation.

G) Custom data - top

You can use custom data to populate the timeline. You can create your own web service or deliver your data in json format. For json data format see demo files. Set the front-end to work with your custom data 
<script type="text/javascript">
$(document).ready(function(){
    var options = {
        basePath : "....",
        feeds:[
            {type : "custom" , id: "/your-data-source/data.json"}
        ]
     
    };
    $(".my-timeline-container").socialmix(options);
});
</script>
If you want to use Javascript Object instaed of JSON datas you can set "id" with variable name. For json data format see demo files. 
<script type="text/javascript">
$(document).ready(function(){
    var js_data = [{ ... }, { ... } ];//object list
    var options = {
        basePath : "....",
        feeds:[
            {type : "custom" , id: "js_data"}
        ]
     
    };
    $(".my-timeline-container").socialmix(options);
});
</script>

H) Skin - top

Socialmix has three different skins. You can find it in "socialmix/css/skins/". To use a skin, load skin_x.css in your html document.

If you want to create your own skin, you can modify an existing skin or create a new one.
